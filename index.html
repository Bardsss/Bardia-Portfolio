Power Apps SharePoint Power Automate

CTA Weekly Reporter

Problem: Weekly field reporting needed clean ServiceNow context and accurate leadership routing without manually maintaining administration lists or reporting hierarchies.

Solution: Designed a structured Power Apps reporting workflow backed by automated scripts that pull ServiceNow information from Azure Blob storage, then verify routing and hierarchy details with a separate Power Automate flow that retrieves organizational data.

Impact: Reduced manual administration management, kept reporting aligned with current org structure, and created cleaner data for executive-style Power BI insights.

Security/GRC Relevance: Stronger auditability, controlled data entry, traceable reporting, and reduced human-error risk in hierarchy-dependent reporting.

Power Apps Outlook Automation

Outage Communication Application

Problem: Service degradation notices took too long to draft and had to stay consistent across repeated outage patterns, ServiceNow PRB details, and company-wide communication needs.

Solution: Built a Power Apps workflow with a custom Copilot Studio agent that reads previous user submissions and drafts the service degradation message from the provided ServiceNow PRB description. Added toggleable autocomplete and auto-description features that reuse prior entries to speed up drafting.

Impact: Reduced degradation notice production time from about 46 minutes to about 8 minutes while improving consistency and reducing repetitive writing.

Security/GRC Relevance: Standardized outage communication, preserved clearer submission history, and integrated a second Copilot Studio French translation agent for company-wide outage messaging.

ServiceNow Power Automate Copilot Studio

ServiceNow Incident AI Initiation

Problem: Newly assigned incidents can sit too long before the requester receives a clear first response, which slows cycle time and creates inconsistent communication.

Solution: Built a Power Automate flow that triggers when an incident is assigned to me, extracts the key ticket context, sends it to a custom Copilot Studio agent, retrieves the generated response, and sends the initial message to the user on my behalf.

Impact: Improves first-touch speed, standardizes user updates, and supports better incident cycle-time metrics through faster initiation and cleaner handoff context.

Security/GRC Relevance: Creates a repeatable communication trail while keeping analyst oversight in the incident workflow.
